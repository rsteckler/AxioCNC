What You Get

AxioCNC solves the problems that crash machines and waste workpieces.

It Won't Crash

Other software crashes in the middle of jobs. AxioCNC keeps running. It's built on the same reliable foundation that's been running production machines for years. Your workpiece stays safe.

You Won't Hit Wrong Buttons

Controls are spaced so you don't accidentally tap the wrong button on a touchscreen. No more crashes from hitting "Z down" when you meant "X+". The interface is designed to prevent costly mistakes.

Works Across Your Network

Design on your main computer. Upload and start jobs from your workshop tablet. Monitor progress from your office. One server, accessible from anywhere on your network. No need to stay in the shop.

Keeps Z Calibration

No bugs that lose Z zero in the middle of a job. Your tool stays where it should be. Touch plate probing works consistently. You can trust the readings.

Use a Joystick

Plug in any USB gamepad or joystick and jog your machine smoothly. Analog control for precise positioning. No more awkward arrow key controls.

Know Which Tool to Grab

Instead of "T4", see "1/4 inch flat endmill". The tool library shows tool names, diameters, and types. You know exactly which tool to pull from your toolbox.

How AxioCNC Came To Be

I started with Carbide Motion but quickly outgrew it. I tried cncjs and it worked great for many years. There were always some gaps I wished were fixed: native joystick support, better UI design (the Z-down button is RIGHT next to the X+ button, and I've crashed my machine many times when my finger tapped the wrong spot on the touch screen), but more than anything, it was abandoned and I wanted to work with a project that was still improving and being maintained.

I went on a search for new sender software. I tried UGS, but the Java app would crash on my Windows machine every few minutes. It crashed 6 times just while I was setting it up, so I never had the confidence to run a job, fearing it would crash in the middle of a workpiece.

I tried gSender, which has a fantastic UI and feature set, and is actively maintained. But I ran into a bug on my first project where the software lost Z calibration in the middle of a job and plunged into my carving.

I tried ioSender and it was incredibly slow and the UI felt overloaded.

Finally, I went back to cncjs and decided I would fix the things that bothered me. Unfortunately, cncjs isn't actively integrating fixes, so I had to create a new version. That started with some minor changes, but has evolved enough that it looks like totally different software now. I focused almost entirely on the experience of the software, combining that with the flawless reliability that cncjs has always delivered.

The result is a G-code sender that is unique in the way it integrates with real-world workflows, while maintaining the benefits of a reliable, networked server. Model and upload from your primary machine, setup and run jobs from the workshop, then monitor progress in the quiet of a separate office.

The experience has been simplified, removing many unused and half-complete features. It's also been expanded to include native joystick support, analog continuous jogging both with joystick and in the UI, native camera support, a tool library to make tool selection simple, a modern 3d visualizer that includes project outline, zeroing methods and strategies, theming support for light and dark mode, and an intuitive task-based dashboard that focuses you on the right tasks when you're setting up a job, monitoring a running job, and viewing the results of your projects.
